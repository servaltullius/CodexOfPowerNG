# Codex of Power NG v1.0.8-rc.5 (2026-02-23, Pre-release)

## 요약
이번 프리릴리즈는 설정 저장 워커가 오래된 메모리 상태를 다시 덮어쓸 수 있던 경로를 차단한 안정화 핫픽스입니다.

## 주요 변경점
- settings 저장 워커의 디스크 저장 경로를 snapshot 전용 API(`SaveSettingsSnapshotToDisk`)로 전환
- Config 레이어에 in-memory 설정을 변경하지 않는 snapshot 저장 API 추가
- 저장/롤백 관련 회귀 테스트를 보강해 stale overwrite 재발 방지 검증 강화

## 사용자 영향
- 기능 동작 변화 없음.
- 드문 타이밍에서 설정 저장 후 이전 값이 다시 보이는 위험이 줄어듭니다.
- 저장 실패 시 롤백 경로의 신뢰도가 향상됩니다.

## 기술 변경 상세
- `include/CodexOfPowerNG/Config.h`: snapshot 저장 API 선언 추가
- `src/Config.cpp`: `SaveSettingsSnapshotToDisk` 구현 추가(Clamp 후 디스크 저장)
- `src/PrismaUISettings.cpp`: 저장 워커 persist 호출을 snapshot API로 변경, 예외 로그 문구 정합화
- `tests/config_atomic_save.test.cjs`: snapshot API가 전역 in-memory 상태를 덮어쓰지 않는 경로 검증 추가
- `tests/prismaui_settings_save_rollback.test.cjs`: 저장 워커가 snapshot 저장 경로를 사용하는지 검증 추가
