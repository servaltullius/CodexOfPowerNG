# Codex of Power NG v1.0.9 (2026-02-25)

## 요약
v1.0.9 정식 릴리즈는 1.0.9-rc.1 ~ rc.3에서 검증한 보상 동기화/저장 안정화 패치를 통합한 버전입니다.
이번 릴리즈의 목표는 **보상 수치 정합성(특히 Carry Weight)**, **로드 경계 안전성**, **UI 복원력**을 함께 개선하는 것입니다.

## 주요 변경점
- **보상 채널 정비(핵심)**: 보상 적용 경로를 `ModActorValue`(임시 채널)에서 `RestoreActorValue(kPermanent)`(영구 채널) 중심으로 정리해, 장비/버프 노이즈와 분리된 동기화가 가능해졌습니다.
- **Carry Weight 과잉 적용 차단**: carry weight 동기화 기준을 `permanentModifier` 중심으로 단순화해, 보상 표시와 실제 적용량이 어긋나는 문제를 줄였습니다.
- **로드 반복 누적 방지**: base-sticky AV(`Health`, `Magicka`, `Stamina`)에서 persistent 근거가 없을 때 양수 재적용을 억제하도록 가드를 강화했습니다.
- **Reward Sync fail-closed 전환**: 메인 task scheduler 불가 시 동기 fallback 루프를 제거하고 rerun 예약 중심으로 동작하도록 바꿨습니다.
- **설정 저장 복구 강화**: 저장 실패 시 현재 메모리 스냅샷이 아니라 마지막 디스크 반영 스냅샷으로 롤백해 `settings.user.json`과 런타임 상태 불일치 위험을 줄였습니다.
- **UI self-heal 보강**: 메인 큐 불가로 갱신이 드롭되면 pending refresh를 기록하고, 다음 JS 요청 시 상태/목록/보상/Undo 패널을 자동 재동기화합니다.

## 사용자 영향
- Carry Weight 포함 보상 수치의 체감 정합성이 개선되어, 표시값과 실제 능력치 차이가 줄어듭니다.
- 로드/전환 경계에서 보상 동기화가 더 보수적으로 동작해 프리징/히치 가능성이 낮아집니다.
- 일시적인 큐 불가 상황 이후 UI 패널이 다음 요청에서 자동 회복되어 stale 화면이 오래 남는 경우가 줄어듭니다.
- 이미 과누적된 세이브는 **Rewards 탭 → Refund** 1회 실행 후 저장/재시작을 권장합니다.
