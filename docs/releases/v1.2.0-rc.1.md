# Codex of Power NG v1.2.0-rc.1 (2026-03-11, Pre-release)

## 요약
이번 프리릴리즈는 기존의 랜덤 누적 보상 축을 `기록 기반 빌드 조립` 구조로 전환하는 첫 공개 빌드입니다. 등록한 아이템이 `공격 / 방어 / 유틸` 기록 점수로 재해석되고, 해금과 활성화를 분리한 새 진행 구조 위에서 Quick Register, Build 화면, 세이브 마이그레이션, 네이티브 브리지가 함께 바뀝니다.

## 주요 변경점
- **빌드 시스템 전환**: 등록 진행이 더 이상 랜덤 보상 누적에 머물지 않고, `Attack / Defense / Utility` discipline score를 쌓아 해금한 옵션을 슬롯에 골라 활성화하는 구조로 바뀌었습니다.
- **해금과 활성화 분리**: 성장 자체는 누적되지만 실제 적용되는 큰 옵션은 활성 슬롯으로 관리해, 특정 옵션 과성장이나 후회 비용을 줄이도록 방향을 바꿨습니다.
- **Quick Register 재설계**: 등록 화면이 계통별 섹션을 가진 배치 등록 흐름으로 바뀌었고, 보호된 항목도 비활성 행으로 남겨 이유를 직접 보여줍니다.
- **네이티브/세이브 마이그레이션**: 기존 reward totals 기반 세이브는 등록 이력을 기준으로 build score를 재구성하고, 레거시 reward 경로 정리를 단계적으로 마무리하도록 전환했습니다.
- **레거시 reward UI 정리**: 보상 cadence/multiplier 성격의 구형 설정은 더 이상 PrismaUI에서 전면 노출하지 않고, build 중심 진행 UX에 맞춰 설정 표면을 정리했습니다.

## 사용자 영향
- 이제 등록을 많이 할수록 무작위 보너스가 쌓이는 대신, 내가 해금한 빌드 옵션 중 무엇을 실제로 활성화할지 직접 선택하게 됩니다.
- 방어력처럼 한 축이 과도하게 올라가는 상황에서는 전체 리셋 대신 활성 옵션 교체만으로 조정할 수 있습니다.
- Quick Register에서 퀘스트 보호/즐겨찾기 보호 같은 이유가 바로 보여서, 왜 체크가 안 되는지 UI 안에서 확인할 수 있습니다.
- 계통별 섹션과 배치 등록 요약이 들어가면서 대량 인벤토리에서도 `이번 등록으로 어느 점수가 오르는지`를 더 읽기 쉬워졌습니다.
- 이번 프리릴리즈는 밸런스와 UX 방향 전환이 크기 때문에, 기존 stable과 비교했을 때 플레이 감각 차이가 분명하게 느껴질 수 있습니다.

## 호환성/주의사항
- Codex of Power / SVCollection과 저장 호환되지 않습니다.
- 기존 Codex of Power NG 세이브는 build migration 경로를 타도록 설계되어 있지만, 이번 프리릴리즈는 진행 구조 자체가 크게 바뀌는 빌드라서 기존 reward 감각과는 차이가 납니다.
- 퀘스트 보호/즐겨찾기 보호 표시가 보강되었지만, 실제 Skyrim 런타임 조합에서의 체감 검증은 여전히 중요합니다.
- 요구사항은 기존과 동일합니다:
  - SKSE64
  - Address Library for SKSE Plugins
  - Prisma UI
  - Media Keys Fix SKSE
