# Codex of Power NG v1.2.0 (2026-03-22)

## 요약
v1.2.0 정식 릴리즈는 `v1.2.0-rc.1`과 `v1.2.0-rc.2`에서 전개한 build-system 전환 트랙을 안정판으로 확정하는 버전입니다. 기존 기록형 progression을 `Attack / Defense / Utility` 기반 빌드 조립 흐름으로 완전히 옮기고, 확장된 build catalog, 점수 기반 슬롯 성장, 카탈로그 중심 Build UI, 그리고 release-track 밸런스 조정까지 한 묶음으로 정식 반영합니다.

## 주요 변경점
- **빌드 시스템 정식 반영**: discipline별 점수 축적, 해금, 활성 슬롯 선택, 영구 빌드 진행도를 중심으로 progression 구조를 재정의했습니다.
- **레거시 기록 마이그레이션 유지**: 기존 등록 이력에서 build score를 재구성하는 마이그레이션 경로를 유지해, Codex of Power NG 저장 라인 내부에서는 새 빌드 구조로 이어집니다.
- **빌드 카탈로그 확장 확정**: 공격/방어/유틸 전반에 저항 번들, 유틸 modifier, resource sustain 계열을 포함한 더 넓은 runtime-backed 옵션 구성이 정식 반영됩니다.
- **점수 기반 슬롯 성장 확정**: slotted option이 discipline build point에 따라 단계적으로 강화되며, 현재 효과와 다음 단계 효과를 Build 화면에서 직접 읽을 수 있습니다.
- **Build UI 재구성 확정**: discipline rail, theme tab, focused option detail rail, grouped batch registration, 1행 활성 슬롯 요약 카드까지 포함한 catalog-first 탐색 구조를 정식 반영했습니다.
- **밸런스 재조정 확정**: weighted build point, unlock pacing, slot competition, hybrid와 specialist 효율 차이를 RC 트랙 피드백 기준으로 재조정했습니다.

## 사용자 영향
- 플레이어는 이제 단순 누적 보상보다 `어떤 discipline에 투자했고 어떤 옵션을 슬롯에 넣었는지`가 더 직접적으로 build 정체성을 결정하는 흐름을 경험하게 됩니다.
- Build 화면에서 카탈로그 탐색, 현재 선택 옵션 확인, 활성 슬롯 훑기, 등록 후보 확인까지의 동선이 짧아져 실제 조립/비교가 더 수월해집니다.
- `v1.1.0` 대비 gameplay와 UI 읽기 방식이 크게 달라진 릴리즈라, 새 빌드 구조와 밸런스 템포에 맞춰 플레이 감각이 달라질 수 있습니다.
- Codex of Power NG 저장 라인 내부 마이그레이션은 유지되지만, Codex of Power / SVCollection과 저장 호환되지 않는 정책은 그대로입니다.

## 호환성/주의사항
- Codex of Power / SVCollection과 저장 호환되지 않습니다.
- 이번 정식 릴리즈는 `v1.1.0` 대비 progression 구조, Build UI, 밸런스, 해금 흐름이 크게 달라지는 기능 릴리즈입니다.
- PrismaUI 패널 밀도와 Build slot summary는 자동 테스트와 패키지 검증 기준으로 확인했지만, 실제 Skyrim 런타임에서의 최종 체감/가독성은 계속 중요합니다.
- 요구사항은 기존과 동일합니다:
  - SKSE64
  - Address Library for SKSE Plugins
  - Prisma UI
  - Media Keys Fix SKSE
