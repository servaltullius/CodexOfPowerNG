# Build Catalog Curation Design

## Summary

현재 빌드 카탈로그는 `공격 / 방어 / 유틸` 각 3개씩의 최소 옵션만 제공한다. 이 구조 자체는 1차 버전으로 충분하지만, 일부 옵션은 계통 정체성이 약하고 설명이 모호하다. 특히 `가속 / 강건 / 정찰`은 설명만 다듬는 수준으로는 해결되지 않는다.

이번 1차 개편의 목표는 옵션 수를 늘리는 것이 아니라, 현재 `3 x 3` 구조를 명확하고 납득 가능한 형태로 고정하는 것이다. 계통별 옵션 수 확장은 이후 UI 밀도와 탐색 방식까지 같이 검토한 뒤 진행한다.

## Goals

- 현재 `공격 / 방어 / 유틸` 3개 구조를 유지한다.
- 설명이 모호한 옵션을 제거하거나 교체한다.
- 남는 옵션은 플레이어가 효과를 즉시 이해할 수 있는 문구로 정리한다.
- 계통별 역할 중복을 줄인다.
- 레거시 랜덤 보상 풀 전체를 즉시 이관하지 않는다.

## Non-Goals

- 계통당 옵션 수를 5개 이상으로 늘리지 않는다.
- 슬롯 수나 해금 점수 체계를 바꾸지 않는다.
- UI 탐색 구조를 다시 설계하지 않는다.
- 레거시 보상 전체를 새 빌드 카탈로그로 1:1 이관하지 않는다.

## Current Problems

### 1. 옵션 수가 적은 것은 의도된 1차 상태다

현재 카탈로그는 [BuildOptionCatalog.cpp](/home/kdw73/Codex%20of%20Power%20NG/src/BuildOptionCatalog.cpp)에 정의된 9개 옵션만 사용한다. 따라서 UI에 계통별 3개만 보이는 것은 현재 설계상 정상이다.

### 2. 설명만 애매한 것이 아니라 효과 설계 자체가 약한 옵션이 있다

- `가속 / Momentum`
  - 현재 설명은 "연속 공격의 유지 타이밍이 넉넉해집니다."
  - 플레이어가 실제 체감 효과를 바로 이해하기 어렵다.
- `강건 / Stalwart`
  - 현재 효과는 `소지 한도 +25`
  - 유틸의 `비축`과 사실상 역할이 겹친다.
- `정찰 / Scout`
  - 현재 설명은 "탐험 중 주변 전리품을 더 쉽게 찾아냅니다."
  - 구현/체감 방식이 불명확하다.

### 3. 설명 문구의 명시성이 부족하다

`맹공`, `정밀`, `수호`, `보루`, `흥정` 같은 옵션은 큰 방향은 맞지만, 일부는 정확히 무엇이 얼마나 변하는지 한 번에 읽히지 않는다.

### 4. 구현 검증 리스크가 있다

코드 검토상 `activeBuildSlots`는 저장되고 UI payload에도 노출되지만, slotted option effect가 실제 런타임 actor value 또는 시스템 효과로 적용되는 경로는 이번 검토 범위에서 명확히 확인되지 않았다. 따라서 구현 전에 "현재 slotted effect가 실제로 적용되는가"를 먼저 검증해야 한다.

## Chosen Approach

`3 x 3` 구조를 유지한 채, 애매한 3개 옵션만 교체하고 나머지 6개는 설명을 명확하게 다듬는 `직관형 재큐레이션`으로 간다.

이 접근을 선택한 이유:

- 현재 UI 밀도와 탐색 구조를 그대로 유지할 수 있다.
- 계통 정체성을 빠르게 또렷하게 만들 수 있다.
- 레거시 보상 전체 이관보다 리스크가 작다.
- 플레이어가 실제로 이해하기 쉬운 빌드 선택지를 먼저 확보할 수 있다.

## Copy Rules

모든 설명 문구는 다음 규칙을 따른다.

- 먼저 `언제 적용되는지`를 쓴다.
  - 예: `슬롯 활성 시`
- 다음에 `무엇이 얼마나 변하는지`를 쓴다.
  - 예: `공격 피해 +5%`
- 추상적 표현을 피한다.
  - 예: `넉넉해집니다`, `더 쉽게`, `유리해집니다` 같은 표현은 가능한 한 수치/행동 기준으로 바꾼다.

### Example Korean Copy Style

- `슬롯 활성 시 공격 피해가 5% 증가합니다.`
- `슬롯 활성 시 무기 공격 속도가 3% 증가합니다.`
- `슬롯 활성 시 최대 체력이 15 증가합니다.`
- `슬롯 활성 시 이동 속도가 3% 증가합니다.`

## Effect Mapping

1차 카탈로그 기준 권장 effect mapping:

- `맹공`
  - `attack_damage_mult`
- `정밀`
  - `weapon_speed_mult`
- `급소`
  - 기존 레거시 보상에 있던 `critical chance` 계열 재사용
- `수호`
  - `damage_resist`
- `보루`
  - `block_percent`
- `인내`
  - `health`
- `비축`
  - `carry_weight`
- `흥정`
  - `vendor_price_bonus`
- `기동`
  - 기존 레거시 보상에 있던 `speed_mult` 계열 재사용

## Open Risk

slotted option이 실제 플레이 효과로 적용되는 런타임 경로가 불명확하다. 따라서 다음 구현은 반드시 아래 순서로 진행해야 한다.

1. 현재 slotted effect 적용 경로 존재 여부를 검증한다.
2. 경로가 있으면 카탈로그/문구만 교체한다.
3. 경로가 없으면 단순 카탈로그 수정이 아니라 `build effect application` 자체를 구현 범위에 포함시킨다.

## Follow-up

1차 카탈로그가 안정화되면, 그 다음 단계에서만 아래를 검토한다.

- 계통당 옵션 수 확장
- 계통별 5~7개 구조
- UI 카드 밀도 조정
- 필터/접기/티어 구분 등 확장 탐색 UX
